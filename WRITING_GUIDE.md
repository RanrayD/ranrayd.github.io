# ✍️ Ranray 的博客写作指南

这份文档是为了帮助你快速上手博客写作和发布流程。

## 1. 写作准备

所有的博客文章都存放在 `_posts` 文件夹中。

### 步骤
1. 打开 `_posts` 文件夹。
2. 复制文件夹里已有的 `.md` 文件，然后重命名。
   - **命名规则**：`YYYY-MM-DD-标题英文.md`
   - 例如：`2025-12-16-my-first-post.md`
   - 注意：日期必须是今天或过去的时间，未来的日期默认不会显示。

### 文章格式
打开你新建的文件，修改顶部的 "头信息" (Front Matter)：

```yaml
---
layout: post
title:  "这里写中文标题"
date:   2025-12-16 12:00:00 +0800
categories: [随笔]
tags: [生活, 记录]
---

这里开始写正文内容...
```

- `title`: 文章显示的标题。
- `date`: 发布时间，保持格式，只需要改日期。
- `categories`: 分类，用于归档。
- `tags`: 标签，用于搜索。

---


## 2. 手动发布 (如果你想学 Git)

如果你想手动操作，可以在文件夹空白处右键选择 "Open Git Bash here" 或在终端中依次输入以下三条命令：

```bash
# 1. 把所有修改过的文件加入暂存区
git add .

# 2. 确认提交，并附带说明信息
git commit -m "写一些更新说明"

# 3. 推送到 GitHub 服务器
git push
```

## 3. 常见问题

- **文章没显示？** 
  - 检查文件名里的日期是不是未来的时间。
  - 检查文件头部的 `date` 格式是否正确。
  - 等待几分钟，GitHub Pages 构建需要时间。

- **图片怎么放？**
  - 把图片放入 `assets` 文件夹。
  - 在文章里这样引用：`![图片描述](/assets/图片名.jpg)`
